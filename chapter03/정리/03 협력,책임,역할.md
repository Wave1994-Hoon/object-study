## 흐름
```
- 객체들이 어플리케이션의 기능을 구현하기 위해 상호작용 하는 것을 `협력`이라고 함
- 객체가 협력에 참여하기 위해 수행하는 로직을 `책임`이라고 함
- 책임들이 모여 수행하는 `역할`을 구성한다.
```

## 객체 
```
- 객체란, 상태와 행동을 함께 캡슐화하는 실행 단위
- 어떤 어플리케이션안에 어떤 객체가 필요하다면 그 이유는 무조건 단 하나여야 한다.
    - 그 객체가 어떤 협력에 참여하고 있기 떄문
```

## 협력
```
- 객체지향 시스템은 `자율적`인 객체들의 공동체이다.
- 협력은 객체지향의 세계에서 기능을 구현할 수 있는 `유일한` 방법
- 객체 사이의 협력을 사용할 때 메시지를 전송하여 커뮤니케이션을 수행
- 즉, 협력이란 어떤 객체가 다른 객체에서 `메시지를 전송`하는 것이다.
```

## 책임
```
- 협력에 참여하기 위해 객체가 수행하는 행동을 의미
- 객체가 유지해야 되는 정보와 수행할 수 있는 행동에 대해여 대략적으로 서술
- 개게의 책임을 크게 하는 것과 아는 것으로 분류 가능
    - 하는 것
        - 객체를 생성하거나 계산을 수행하는 등의 스스로 하는 것
        - 다른 객체의 행동을 시작시키는 것
        - 다른 객체의 행동을 제어하고 조절하는 것
    - 아는 것
        - 사적인 정보에 관해 아는 것
        - 관련된 객체에 관해 아는 것
        - 자신이 유도하거나 계산할 수 있는 것에 관해 아는 것 
- 예제
    - Screening 객체의 책임은 영화를 예매하는 것
        - 자신이 상영할 영화를 알고 있어야 한다. -> 아는 것과 관련된 책임 
    - Movie 객체의 책임은 요금을 계산하는 것 
        - 예매 가격을 계산할 책임 -> 하는 것과 관련된 책임 
```


## 역할
```
- 책임의 집합을 의미
- 역할은 유연하고 재사용 가능한 협력을 얻을 수 있다.
- 동일한 책임을 수행하는 역할을 기반으로 협력을 하나로 통합 가능 
- 예시
    - AmountDiscountPolicy / PercentDiscountPolicy 존재
    - 각각을 구현하면 중복되는 코드가 발생
    - 할인 요금을 계산하는 DiscountPolicy를 만들어서 추상화 
```
